# Security Policy

## Supported Versions

Security fixes are provided for the latest stable release published to Maven Central.

| Version | Supported          |
| ------- | ------------------ |
| 1.5.x   | :white_check_mark: |
| < 1.5   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly — **do not open a public GitHub issue**.

### Preferred: GitHub Security Advisories

Use [GitHub's private vulnerability reporting](https://github.com/santimattius/kmp-resilient/security/advisories/new) for this repository. This keeps details confidential until a fix is available.

### What to Include

- A description of the vulnerability and its potential impact
- Steps to reproduce the issue
- Affected versions and platforms (JVM, Android, JS, Native, etc.)
- Any suggested fix or mitigation, if you have one

### What to Expect

- **Acknowledgment** within 7 days of your report
- **Status update** on investigation and remediation timeline
- **Coordinated disclosure** — we will work with you on a reasonable disclosure date after a fix is released

Thank you for helping keep Resilient and its users safe.
